# Demo Video Storyboard

Purpose: post-MVP product demo video for enterprise buyers  
Recommended length: 90-150 seconds  
Primary message: realistic AI-assisted engineering assessment with evidence-backed reports

## Production Notes

- Record after MVP verification, using a prepared local session with realistic telemetry.
- Keep the story to three steps: Create a Challenge, Share One Link, Get AI-Scored Reports.
- Use actual product routes and UI, not mock screens.
- Avoid claims about production SSO, RBAC, sandbox isolation, compliance certification, or ATS integrations.
- If showing local URLs, use them as demo environment URLs, not production URLs.

## On-Screen Rubric Caption

Use this once during report shots:

`ai-collaboration-v1`: Problem Decomposition, First-Principles Thinking, Creative Problem Solving, Iteration Quality, Debugging with AI, Architecture Decisions, Communication Clarity, Token Efficiency.

## Suggested Narration

"Traditional coding tests tell you whether a candidate can solve a narrow exercise. Hirewave shows how they work when AI is part of the engineering workflow. Create a challenge, share one candidate link, and review an AI-scored report backed by prompts, commands, tests, edits, and final files. The result is not just a score. It is a defensible hiring conversation grounded in evidence."

## Capture Checklist

- A created assessment exists.
- Candidate invite link opens successfully.
- Candidate session has at least one AI message.
- Candidate session has at least one command/test run.
- Candidate session has a submitted report.
- Report includes dimension scores, evidence, transcript, commands/tests, final files, and next interview focus.
- Any visible candidate names and emails are demo-safe.
- Any limitation callouts are stated as current MVP limitations, not hidden.
